`@dudousxd/nestjs-agent` is a monorepo of small, single-purpose packages. [`-core`](/docs/agent/packages/core)
defines the SPIs and the agentic loop — the **mechanism**. Everything else either implements one of
those SPIs (a model adapter, a store, a dashboard) or consumes the library from an app (the NestJS
module, the React frontend). Which tables, which tenant column, which roles, which model — that's
**policy** you supply by installing the right package and configuring it, never by forking the loop.

## The map

| Package | Role |
|---|---|
| [`-core`](/docs/agent/packages/core) | Framework-agnostic agent loop, tool registry, and the SPIs (`ModelProvider`, `AgentStore`, `AgentGovernanceQueries`, `ActorResolver`, `RolesPolicy`, `TokenStreamSink`, `QuotaStore`). |
| [`(umbrella)`](/docs/agent/packages/nestjs) | NestJS `AgentModule` + `@AiTool` + the `/agent` controllers; the `/durable` subpath for the durable runner. |
| [`-ai-sdk`](/docs/agent/packages/ai-sdk) | `aiSdkModel(model)` adapts any Vercel AI SDK v7 `LanguageModel` to `ModelProvider` — zero provider code. |
| [`-react`](/docs/agent/packages/react) | `useAgentChat` + styling-agnostic chat components for a browser frontend. |
| [`-data`](/docs/agent/packages/data) | `createExecuteSqlTool` — governed, read-only SQL as a prebuilt tool. |
| [`-rag`](/docs/agent/packages/rag) | Retrieval-Augmented Generation — chunking, ingestion, an embedding-backed `Retriever`, and in-memory + pgvector stores. |
| [`-rag-media`](/docs/agent/packages/rag-media) | Auto-ingest `nestjs-media` uploads into agent RAG — extract, chunk, embed, index, owner-scoped, delete-synced. |
| [`-dashboard`](/docs/agent/packages/dashboard) | Standalone AI-gateway governance console (bundled SPA + API), no Telescope required. |
| [`-store-mikro-orm`](/docs/agent/packages/store-mikro-orm) | `AgentStore` + `AgentGovernanceQueries` + `AgentPricingStore` on MikroORM. |
| [`-store-drizzle`](/docs/agent/packages/store-drizzle) | `AgentStore` + `AgentGovernanceQueries` + `AgentPricingStore` on Drizzle. |
| [`-transport-redis`](/docs/agent/packages/transport-redis) | `RedisTokenStreamSink` — a `TokenStreamSink` over Redis for multi-replica deployments. |
| [`-telescope`](/docs/agent/packages/telescope) | Telescope extension — an "Agent" tab reusing the governance read-model. |
| [`-authz`](/docs/agent/packages/authz) | Plugs `@dudousxd/nestjs-authz` into tool authorization as a `RolesPolicy`. |
| [`-codegen`](/docs/agent/packages/codegen) | Emits `api.agent.*` into the ecosystem's unified typed client. |
| [`-testing`](/docs/agent/packages/testing) | In-memory store/sink/quota + a deterministic fake model for offline tests. |

## How they layer

```text
-react ──▸ (SSE /agent/chat) ──▸ umbrella (@dudousxd/nestjs-agent) ──▸ -core
-codegen ─▸ (REST /agent/*)  ──┘                                       │
                                                                        │  (SPIs)
        -store-mikro-orm / -store-drizzle ─────────────────────────────┤  AgentStore + AgentGovernanceQueries + AgentPricingStore
                              -ai-sdk ──────────────────────────────────┤  ModelProvider
                              -authz ────────────────────────────────────┤  RolesPolicy
                              -transport-redis ────────────────────────────┤  TokenStreamSink
                              -data (createExecuteSqlTool) ───────────────────┤  a read-kind ToolSpec + ToolHandler
                              -rag ─────────────────────────────────────────────┤  Retriever + EmbeddingProvider (RAG tool / inject)
                              -rag-media (auto-ingest via diagnostics channel) ──┤  feeds -rag's VectorStore, no hard dep
        -dashboard / -telescope ──────────────────────────────────────────────┘  consume AgentGovernanceQueries
```

`-core` has no NestJS dependency and defines every contract; the umbrella package is the only one
that depends on NestJS DI. A store, the AI SDK adapter, and the authz adapter each implement exactly
one SPI. The dashboard and the Telescope extension are peers — both read the same
`AgentGovernanceQueries` read-model and the same `aviary:agent:*` diagnostics channel, so you can run
either, both, or neither.
