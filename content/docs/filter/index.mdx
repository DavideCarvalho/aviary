**Filter** turns query strings into safe, typed database filters. Define what's filterable, and Filter handles parsing, validation and translation to your ORM — with a matching typed client and React hooks for the front end.

<Callout title="Aviary call sign: Sandpiper">
Part of the **Aviary** ecosystem — every library reads from a shared [Context](/docs/context),
so adopting one makes the next one cheaper.
</Callout>

## Install

```bash
pnpm add @dudousxd/nestjs-filter
```

## Two ways to fly

### Plug-n-play

Declare filterable fields on a resource and get a ready endpoint that accepts a structured filter query, validated and translated for you.

### Fully configurable

Choose your ORM adapter (MikroORM or TypeORM), generate a typed client, and drop in the React bindings — or use just the core and translate filters yourself.
