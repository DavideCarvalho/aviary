**Notifications** lets you describe a notification once and deliver it across many channels — mail, database, push, SMS, Slack, Discord, Teams, Telegram, webhooks and real-time (SSE / broadcast). Preferences, delivery tracking and queueing are first-class.

<Callout title="Aviary call sign: Nightingale">
Part of the **Aviary** ecosystem — every library reads from a shared [Context](/docs/context),
so adopting one makes the next one cheaper.
</Callout>

## Install

```bash
pnpm add @dudousxd/nestjs-notifications
```

## Two ways to fly

### Plug-n-play

Define a notification class with the channels it supports, then `notify(user, new InvoicePaid())`. Database + mail work out of the box.

### Fully configurable

Add channels à la carte, pick a database driver (Prisma / TypeORM / MikroORM), queue with BullMQ or Redis, honour per-user preferences, and track deliveries.
