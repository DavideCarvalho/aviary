**`@dudousxd/nestjs-resilience`** is a NestJS-native resilience toolkit — composable **timeout**, **retry**, **circuit breaker** and **failover** policies, with a pluggable, distributed circuit-breaker store. Think *cockatiel for NestJS*, wired into the ecosystem's conventions ([diagnostics](/docs/diagnostics), [context](/docs/context), event-emitter).

<Callout type="info" title="Alpha">
Published as alpha at `0.2.0`. The API here matches the current source. Pin a version in production.
</Callout>

## Compose policies, outer to inner

Each policy is a small object with `execute(fn)`. `wrap()` composes them — the first argument is the outermost layer:

```ts
import { wrap, timeout, retry, circuitBreaker, InMemoryResilienceStore } from '@dudousxd/nestjs-resilience';

const policy = wrap(
  timeout(2_000),
  retry({ attempts: 3, backoff: exponential() }),
  circuitBreaker({ key: 'payments', store: new InMemoryResilienceStore(), threshold: 5, cooldownMs: 30_000 }),
);

await policy.execute(() => chargeCard(order)); // timeout → retry → breaker
```

Three ways to use it: the **programmatic** policies above, the **`@Timeout` / `@Retry` / `@CircuitBreaker` decorators** (wrapped at startup by a `DiscoveryService` explorer), and an injectable **`ResilienceService`** via `ResilienceModule.forRoot()`.

## Ecosystem integration

State transitions (circuit opened/closed/half-open, short-circuited, timeout, retry, failover) emit over [`@dudousxd/nestjs-diagnostics`](/docs/diagnostics) on the `aviary:resilience:*` channel — so [Telescope](/docs/telescope) or any subscriber can observe them. An optional `@nestjs/event-emitter` mirror lets you react in-app, and [`@dudousxd/nestjs-context`](/docs/context) makes breaker keys tenant-aware automatically.

## Packages

| Package | Description |
| --- | --- |
| `@dudousxd/nestjs-resilience` | The policy engine (timeout/retry/circuit-breaker/wrap/failover), the `ResilienceStore` contract, an in-memory store, and the NestJS surfaces (module, service, decorators). |
| `@dudousxd/nestjs-resilience-store-redis` | Distributed circuit-breaker store on Redis (atomic Lua admit/record). |
| `@dudousxd/nestjs-resilience-store-typeorm` | Distributed store on Postgres via TypeORM (`SELECT … FOR UPDATE`). |
| `@dudousxd/nestjs-resilience-store-mikro-orm` | Distributed store on Postgres via MikroORM. |
| `@dudousxd/nestjs-resilience-store-prisma` | Distributed store on Postgres via Prisma. |
| `@dudousxd/nestjs-resilience-store-drizzle` | Store on SQLite via Drizzle (better-sqlite3). |

Every store passes the same shared contract (`runResilienceStoreContract`, exported from `@dudousxd/nestjs-resilience/testing`) — the distributed ones are validated against real engines with testcontainers, including the concurrency invariant: exactly one half-open probe under load.

## Quickstart

<Steps>
<Step>

Install the core. Add a distributed store only if you want breaker state shared fleet-wide:

```bash
pnpm add @dudousxd/nestjs-resilience
# fleet-wide breaker state?
pnpm add @dudousxd/nestjs-resilience-store-redis ioredis
```

</Step>
<Step>

Wrap any flaky call with a composed policy, or reach for `failover()` to try an ordered list of targets until one succeeds:

```ts
import { failover, wrap, timeout, circuitBreaker } from '@dudousxd/nestjs-resilience';

const send = failover({
  targets: providers,
  run: (p) => p.send(payload),
  policy: (p) => wrap(circuitBreaker({ key: p.id, store, threshold: 5, cooldownMs: 30_000 }), timeout(8_000)),
});
```

</Step>
<Step>

Observe the state transitions in [Telescope](/docs/telescope) (or any [diagnostics](/docs/diagnostics) consumer) on the `aviary:resilience:*` channel.

</Step>
</Steps>

Source, full API and the store-adapter docs live in the [GitHub repository](https://github.com/DavideCarvalho/nestjs-resilience).
</Callout>
